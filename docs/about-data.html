How the data works

A plain-English walkthrough of 01_download.R — the script that fetches, cleans, and saves all the raw data powering this map.

Environment

Where the code runs

All data preparation for GreenSpacesMap is written in R and runs inside Posit Cloud — a free, browser-based environment that requires no software installation. If you have a Chromebook, a tablet, or any device with a browser, you can run this code.

Posit Cloud (formerly RStudio Cloud)

Founded in 2009, Posit is a public benefit corporation committed to open source data science. Their cloud platform gives anyone access to a full R environment in the browser — no installation, no configuration. Free accounts are available at posit.cloud. The code for this project is forkable at github.com/jacobfarkas/GreenSpacesMap.

Dependencies

R packages used

The download script loads five packages. Each one does a specific job.

tidyverse sf httr h3jsr janitor
01

tidyverse

A collection of R packages for data wrangling. Handles reading CSVs, filtering rows, transforming columns, and summarizing data. Think of it as R's general-purpose toolkit.

02

sf — simple features

Makes R understand spatial data — points, lines, and polygons with real geographic coordinates. Without sf, a park is just a row in a table. With sf, it's a shape R can measure, map, and do math with.

03

httr

Handles web requests — lets R talk to open data APIs the same way a browser loads a webpage, but programmatically. Used to verify API connectivity during development.

04

h3jsr

R's interface to Uber's H3 hexagonal grid system. Generates the hex cells that cover NYC at resolution 9 — roughly one hexagon per city block — used in the scoring pipeline.

05

janitor

Cleans messy column names. MTA data arrives with column names like "GTFS Latitude" (spaces, mixed case). janitor's clean_names() converts everything to snake_case so it's easier to work with in code.

01_download.R — package loading 
library(tidyverse)
library(sf)
library(httr)
library(h3jsr)
library(janitor)

dir.create("data-prep/raw", recursive = TRUE,
           showWarnings = FALSE)

Data sources

What gets downloaded

The script fetches four datasets from open government APIs. All are free, publicly licensed, and require no API key.

262 Neighborhood tabulation areas
2,058 NYC park properties
496 Subway stations
13,572 Active bus stops

2020 Neighborhood Tabulation Areas (NTAs)

NYC Open Data · dataset 9nt8-h7nd · GeoJSON

The 262 neighborhood boundaries used to aggregate scores. Published by the NYC Department of City Planning, based on 2020 Census tracts. These are the polygons that get colored green, yellow, or red on the map.

data.cityofnewyork.us/resource/9nt8-h7nd.geojson

NYC Parks Properties

NYC Open Data · dataset enfh-gkve · CSV with WKT geometry

All 2,058 park properties managed by the NYC Department of Parks & Recreation. Each record includes the park name, borough, acreage, address, and a full polygon geometry stored as Well-Known Text (WKT). The geometry is converted to spatial format using sf.

data.cityofnewyork.us/resource/enfh-gkve.csv

MTA Subway Stations

NY State Open Data · dataset 39hk-dx4f · CSV

All 496 subway and Staten Island Railway stations, published by the MTA. Includes station name, lines served, borough, GPS coordinates, and — critically — ADA accessibility status coded as 0 (not accessible), 1 (fully accessible), or 2 (partially accessible, one direction only).

data.ny.gov/api/views/39hk-dx4f/rows.csv

MTA Current Bus Stops

NY State Open Data · dataset ai5j-txmn · CSV

All active MTA bus stops across NYC. The raw dataset has 22,977 rows — one per route per stop — so stops served by multiple routes appear multiple times. The script deduplicates to 13,572 unique physical locations and filters to revenue stops only (active stops where passengers board and alight). All bus stops are step-free by design.

data.ny.gov/api/views/ai5j-txmn/rows.csv

Accessibility

The subway accessibility data

The MTA's ADA field uses a three-value numeric code. The script translates these into human-readable labels used throughout the map and the AI-generated stories.

Code Label Count What it means
1 Fully accessible 159 Step-free access in both directions via elevator
2 Partially accessible 8 Accessible in one direction only (noted in ADA Direction Notes)
0 Not accessible 329 No step-free path from street to platform

Only 32% of NYC subway stations are fully accessible — a figure that shapes the equity analysis at the core of this project. Bus stops, by contrast, are all step-free.

Output

What the script produces

Each dataset is saved as a GeoJSON file in data-prep/raw/. GeoJSON is an open, human-readable format for geographic data — it can be opened in any mapping tool, inspected in a text editor, and read directly by Leaflet.js in the browser.

Files created by 01_download.R 
data-prep/raw/
├── nta.geojson         # 262 neighborhood boundaries
├── parks.geojson       # 2,058 park polygons
├── subway.geojson      # 496 stations + ADA status
└── bus_stops.geojson   # 13,572 active stop locations

These files are committed to the GitHub repository and serve as the input for 02_score.R, which generates the H3 hexagon grid and calculates park access scores for every neighborhood.

Reproducibility

Run this yourself

The entire data pipeline is open source and reproducible. No API keys, no paid accounts, no proprietary software required.

1

Create a free Posit Cloud account

Go to posit.cloud and sign up. No credit card required.

2

Fork the repository

Visit github.com/jacobfarkas/GreenSpacesMap and click Fork. This creates your own copy of the project.

3

Open in Posit Cloud

Create a new project from your forked GitHub repo. Posit Cloud will pull in all the files automatically.

4

Install packages and run

Install the five packages listed above, then run 01_download.R. The four GeoJSON files will appear in data-prep/raw/ within a few minutes.

All data is public domain. The NYC Open Data and NY State Open Data portals publish datasets under open licenses. The MTA subway and bus data is free to use for non-commercial purposes under the MTA's developer terms. No attribution beyond this page is required to use the data, but crediting the original sources is appreciated.
